Manifesto do Projeto

EndoFlix é um player de vídeos web robusto, construído com Flask e Bootstrap, para organizar e reproduzir arquivos de vídeo de forma prática e dinâmica. Escaneie pastas, crie playlists, salve sessões, randomize vídeos com shuffle (individual ou automático) e aproveite um layout com 4 players simultâneos, tudo integrado com PostgreSQL para gerenciamento de dados.

Linguagens Utilizadas

  • Python: Backend com Flask para servidor web
  • JavaScript: Interatividade e manipulação DOM
  • HTML/CSS: Estrutura e estilização com Bootstrap
  • SQL: PostgreSQL para persistência de dados

Módulos e Bibliotecas

  • Flask: Framework web
  • psycopg2: Conector PostgreSQL
  • Redis: Cache de metadados
  • Pathlib: Manipulação de caminhos
  • Concurrent.futures: Processamento paralelo

Ferramentas

  • FFmpeg: Extração de metadados de vídeo
  • Redis Server: Cache em memória
  • PostgreSQL: Banco de dados relacional
  • Git: Controle de versão

Objetivos

Fornecer uma solução completa para gerenciamento e reprodução de vídeos locais, com interface moderna e funcionalidades avançadas de organização.

Funcionalidades Principais

  • Escaneamento automático de pastas
  • Gerenciamento de playlists e sessões
  • Shuffle inteligente com priorização de vídeos não reproduzidos
  • Interface com 4 players simultâneos
  • Drag-and-drop para carregamento rápido
  • Sistema de favoritos
  • Captura de snapshots e bursts

Abordagem de Design

Design moderno e responsivo focado na usabilidade, com layout em grade 2x2 para players, barra lateral ajustável e tema escuro/claro. Utiliza animações suaves e feedback visual para melhor experiência do usuário.

Análise do Projeto

Pontos Fortes

  • Interface moderna e responsiva com Bootstrap
  • Integração robusta com PostgreSQL e Redis
  • Funcionalidades avançadas como shuffle inteligente e auto-shuffle
  • Suporte a múltiplos formatos de vídeo
  • Sistema de cache eficiente para metadados
  • Drag-and-drop intuitivo
  • Captura de snapshots e bursts

Pontos Fracos

  • Dependência de FFmpeg e Redis instalados localmente
  • Configuração inicial complexa com PostgreSQL
  • Limitação a arquivos locais (não suporta streaming remoto)
  • Possível lentidão com bibliotecas grandes de vídeos
  • Falta de testes automatizados abrangentes

Principais Funções Implementadas

Função que varre diretórios recursivamente em busca de arquivos de vídeo suportados (.mp4, .mkv, .mov, etc.). Utiliza processamento paralelo para extrair metadados com FFmpeg, calcula hashes para detecção de duplicatas e indexa tudo no PostgreSQL. Suporta atualização incremental de pastas já escaneadas.

Sistema de randomização que prioriza vídeos não reproduzidos recentemente. Oferece shuffle geral (4 vídeos aleatórios), shuffles individuais por player e modo auto-shuffle com intervalo configurável. Mantém histórico de reproduções para evitar repetições excessivas.

Permite criação, salvamento e carregamento de playlists no PostgreSQL. Suporta playlists temporárias durante escaneamento e conversão para permanentes. Inclui atualização automática baseada na pasta de origem e remoção de playlists.

Salva e restaura estados de reprodução com os vídeos em cada player. Permite compartilhamento de sessões entre usuários e gerenciamento completo (criar, carregar, remover).

Sistema de captura de imagens estáticas dos vídeos em reprodução. Suporta snapshots únicos e bursts (sequências rápidas). Utiliza canvas HTML5 para captura e salva em formato WebP na pasta do vídeo.

Perguntas Frequentes

Clone o repositório, instale as dependências com pip, configure PostgreSQL criando o banco 'videos', execute o script SQL para tabelas e rode python main.py. Certifique-se de ter FFmpeg e Redis instalados.

O EndoFlix suporta .mp4, .mkv, .mov, .divx, .webm, .mpg e .avi. Utiliza FFmpeg para extração de metadados e reprodução.

O shuffle prioriza vídeos não reproduzidos. Shuffle geral carrega 4 vídeos aleatórios. Shuffles individuais afetam apenas um player. Auto-shuffle repete automaticamente em intervalos configuráveis.

Sim, você pode salvar o estado atual dos 4 players como uma sessão nomeada no PostgreSQL. Sessões podem ser carregadas posteriormente para retomar exatamente de onde parou.

Clique no ícone de câmera 📸 em qualquer player para capturar um snapshot único, ou use 🔄 para burst (sequência rápida). As imagens são salvas automaticamente na pasta do vídeo em formato WebP.
