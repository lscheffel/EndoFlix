EndoFlix

Player de Vídeos Web Robusto e Dinâmico

Manifesto

EndoFlix é um player de vídeos web robusto e moderno, construído com Flask e Bootstrap, projetado para organizar e reproduzir arquivos de vídeo de forma prática e dinâmica. Nossa missão é fornecer uma interface intuitiva e poderosa para gerenciamento de coleções de vídeo, combinando tecnologia avançada com usabilidade excepcional.

Desenvolvido com foco em performance e escalabilidade, o EndoFlix oferece recursos avançados como indexação inteligente de arquivos, streaming otimizado, gerenciamento de playlists e sessões, além de um dashboard analítico completo. Acreditamos que o acesso organizado aos conteúdos visuais deve ser simples, eficiente e agradável.

Arquivos

main.py

Arquivo principal da aplicação Flask contendo todas as rotas, lógica de negócio e integração com banco de dados, Redis e processamento de arquivos.

config.py

Configurações centralizadas do sistema incluindo parâmetros de banco de dados, Redis, caminhos de arquivos e configurações de processamento.

db.py

Gerenciamento de conexões PostgreSQL com pool de conexões, context managers e métodos utilitários para execução de queries e updates.

file_processor.py

Processamento de arquivos de vídeo incluindo cálculo de hash, extração de metadados com FFmpeg e processamento em lote com paralelização.

cache.py

Implementação de cache Redis para metadados de vídeo, configurações de compressão e gerenciamento de TTL para otimização de performance.

snapshot_processor.py

Sistema de captura de snapshots e bursts de vídeo com processamento assíncrono, salvamento em WebP e gerenciamento de workers.

Templates/base.html

Template principal da interface com layout de 4 players, controles de playlist, sessões e funcionalidades de shuffle e drag-and-drop.

Templates/ultra.html

Dashboard analítico avançado com gráficos Chart.js, estatísticas de uso, tabelas de vídeos mais reproduzidos e controles remotos.

Templates/about.html

Página informativa sobre o projeto, tecnologias utilizadas e informações do desenvolvedor.

Templates/keymaps.html

Documentação completa dos atalhos de teclado disponíveis na aplicação para controle rápido dos players.

README.md

Documentação completa do projeto incluindo instalação, uso, funcionalidades, tecnologias e guia de contribuição.

requirements.txt

Lista de dependências Python necessárias para execução do projeto, incluindo Flask, PostgreSQL driver e outras bibliotecas.

Static/

Diretório de arquivos estáticos incluindo favicon, arquivos JavaScript para mapas de teclado e outros recursos web.

Análise

Arquitetura e Tecnologias

EndoFlix é construído com uma arquitetura modular e escalável, utilizando Flask como framework web principal. O sistema emprega PostgreSQL para persistência de dados com pool de conexões otimizado, Redis para cache de metadados e FFmpeg para processamento de vídeo.

Componentes Principais:
  • Backend Flask: API RESTful com roteamento eficiente e tratamento de erros robusto
  • Banco de Dados: PostgreSQL com tabelas para arquivos, playlists, sessões e estatísticas
  • Cache Redis: Armazenamento temporário de metadados e configurações
  • Processamento: Extração de metadados com FFmpeg e processamento paralelo
  • Frontend: Bootstrap 5 com interface responsiva e JavaScript para interatividade
Características Técnicas:
  • Streaming de vídeo com suporte a range requests HTTP
  • Indexação inteligente baseada em hash SHA256
  • Processamento assíncrono com ThreadPoolExecutor
  • Interface drag-and-drop para gerenciamento de vídeos
  • Dashboard analítico com gráficos em tempo real
  • Sistema de snapshots com captura em WebP

Funções

Sistema avançado de escaneamento que varre diretórios recursivamente, identifica arquivos de vídeo suportados (.mp4, .mkv, .mov, etc.), calcula hashes únicos para detecção de duplicatas e extrai metadados completos incluindo codec, resolução, duração e orientação.

Streaming de vídeo com suporte completo a range requests HTTP, permitindo seek eficiente, buffering inteligente e reprodução suave. Suporte a múltiplos codecs e formatos com transcodificação automática quando necessário.

Criação, edição e gerenciamento de playlists dinâmicas. Suporte a playlists temporárias durante escaneamento, salvamento permanente, atualização automática baseada em pastas de origem e compartilhamento de playlists entre sessões.

Salvamento e restauração de sessões completas com configuração de 4 players. Sessões nomeadas com timestamp automático, carregamento rápido e gerenciamento de histórico de visualização.

Sistema de shuffle avançado com três modos: shuffle geral (4 vídeos aleatórios), shuffles individuais por player e auto-shuffle com intervalo configurável. Prioriza vídeos não reproduzidos recentemente.

Marcação de vídeos favoritos com interface visual intuitiva. Playlist dedicada de favoritos, indicadores visuais nos players e integração com sistema de estatísticas.

Sistema de captura de screenshots durante reprodução com dois modos: snapshots únicos e bursts (sequências rápidas). Salvamento automático em WebP com processamento assíncrono em background.

Painel Ultra Mode com estatísticas completas, gráficos de uso de playlists, tipos de arquivo, timeline de sessões e tabela de vídeos mais reproduzidos. Exportação de relatórios em CSV.

Interface moderna com layout 2x2 de players, barra lateral ajustável, drag-and-drop entre lista e players, controles de velocidade individuais, tema escuro/claro e atalhos de teclado completos.

Recurso de emergência que abre sites aleatórios (Google, YouTube, Wikipedia) em nova aba. Configurável com URLs personalizadas e armazenamento local de preferências.

FAQ

Para escanear vídeos, insira o caminho completo da pasta no campo "Caminho da pasta" (ex.: X:\Tiktok ou /home/user/videos). Clique em "Carregar Pasta" para iniciar o processo. O sistema irá varrer recursivamente todos os arquivos de vídeo suportados e indexá-los no banco de dados.

Após carregar vídeos, selecione os arquivos desejados na lista lateral. Clique em "Save Playlist" e dê um nome descritivo. As playlists salvas aparecem no dropdown "Selecionar Playlist". Use "Remove Playlist" para excluir ou "Update Playlist" para sincronizar com alterações na pasta original.

Use "Shuffle" para carregar 4 vídeos aleatórios em todos os players. Para shuffles individuais, use "Shuffle 1", "Shuffle 2", etc. Ative "Auto Shuffle" e configure o intervalo em segundos para reprodução automática contínua. O sistema prioriza vídeos não visualizados recentemente.

Configure os 4 players com os vídeos desejados usando drag-and-drop ou shuffle. Clique em "Save Session" para salvar a configuração atual com nome automático baseado na playlist e timestamp. Use o dropdown "Selecionar Sessão" para carregar sessões salvas anteriormente.

Clique em "Ultra Mode" na barra de navegação para acessar o dashboard analítico. Visualize estatísticas gerais, gráficos de playlists e tipos de arquivo, tabela de vídeos mais reproduzidos e histórico de sessões. Use os controles para exportar relatórios em CSV.

Durante a reprodução, clique no ícone de câmera (📸) em qualquer player para capturar um snapshot único, ou no ícone de refresh (🔄) para bursts. Configure o intervalo e quantidade de shots no painel lateral. Os snapshots são salvos automaticamente na pasta snapshots/ do vídeo.

Verifique se o arquivo não está corrompido e se o codec é suportado (H.264 recomendado). Certifique-se de que o FFmpeg está instalado e configurado corretamente. Verifique os logs no terminal para mensagens de erro específicas. Tente reindexar a pasta se o problema persistir.

Configure adequadamente o pool de conexões PostgreSQL e Redis. Use SSD para armazenamento de vídeos. Ajuste MAX_WORKERS baseado na CPU disponível. Mantenha o cache Redis ativo para metadados. Evite escanear pastas muito grandes de uma vez.

Sugestões

Dicas para Melhor Uso
Organização de Arquivos
  • Mantenha vídeos organizados em pastas lógicas por categoria
  • Use nomes descritivos para arquivos e pastas
  • Evite caracteres especiais nos nomes de arquivo
  • Considere tamanho de arquivo para otimização de storage
Gerenciamento de Playlists
  • Crie playlists temáticas (filmes, séries, documentários)
  • Use nomes descritivos para fácil identificação
  • Aproveite a atualização automática baseada em pastas
  • Faça backup regular das playlists importantes
Otimização de Performance
  • Mantenha Redis ativo para cache de metadados
  • Configure pool de conexões adequadamente
  • Use SSD para melhor performance de I/O
  • Monitore uso de CPU e memória
Recursos Avançados
  • Configure URLs personalizadas para o botão de pânico
  • Use atalhos de teclado para controle rápido
  • Aproveite o sistema de favoritos para vídeos preferidos
  • Explore o dashboard analítico para insights de uso
Manutenção do Sistema
  • Faça backup regular do banco PostgreSQL
  • Mantenha dependências atualizadas
  • Monitore logs para identificar problemas
  • Atualize FFmpeg para suporte a novos codecs
  • Verifique espaço em disco regularmente
